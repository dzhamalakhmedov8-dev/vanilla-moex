# План исправлений и доработок

## Уже сделано перед публикацией

1. Убраны учетные данные из публикуемой части проекта: live-запуск использует `RABBITMQ_URL`.
2. Разделена логика:
   - `trading_core.py` считает заявку и не зависит от сети;
   - `vanilla_bot.py` отвечает за CLI и RabbitMQ;
   - `backtest_moex.py` отвечает за исторические данные и отчет.
3. Добавлен безопасный режим по умолчанию: без `--live` заявка только печатается.
4. Backtest стал воспроизводимым: отчет сохраняет свечи, а повторный запуск можно сделать через `--candles-file`.
5. Добавлены тесты для расчета заявки, ошибок входных данных, fake RabbitMQ и backtest.
6. `.gitignore` настроен так, чтобы не публиковать локальные отчеты, конспекты, PDF, `.venv` и файл задания с паролем.

## Что еще можно улучшить

1. Добавить `config.example.json`, чтобы параметры стратегии и заявки можно было запускать из файла, а не только из CLI.
2. Сделать отдельный режим `paper`, который будет имитировать исполнение заявки на live market-data без публикации в `sandbox.orders`.
3. Расширить backtest:
   - подобрать параметры `fast/slow` по сетке;
   - сравнить несколько инструментов;
   - добавить out-of-sample период.
4. Добавить график equity curve в HTML или PNG, чтобы отчет было проще показать без открытия CSV.
5. Подключить `ruff` или `black` для единого стиля, но не превращать проект в тяжелый фреймворк.
6. Добавить GitHub Actions: установка зависимостей, `unittest`, `py_compile`, `pip check`.
7. Для live-режима добавить более подробный лог: время ожидания market data, order_id, статусы и причины остановки.
8. Добавить отдельный документ с разбором низкой доходности стратегии: трендовость рынка, комиссии, проскальзывание, SMA-запаздывание и выбранный период.

## Приоритет

Самые полезные следующие шаги:

1. GitHub Actions, чтобы друг сразу видел зеленые проверки.
2. График backtest, чтобы результат можно было показать визуально.
3. `paper`-режим, чтобы связать live market-data и безопасную демонстрацию без реальной отправки заявки.
