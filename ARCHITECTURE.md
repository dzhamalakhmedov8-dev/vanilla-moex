# Архитектура Vanilla

Проект состоит из двух рабочих модулей.

## trading_core.py

В этом файле собраны расчёты без сети:

- чтение параметров `SBER.json` и `CNYRUB_TOM.json`;
- проверка поддерживаемого инструмента;
- выбор стороны заявки по `qty` или `volume`;
- выбор способа цены с приоритетом `price > slippage > best_quote`;
- округление цены по `MINSTEP`;
- перевод суммы в рублях в целые лоты;
- сборка JSON-заявки для `sandbox.orders`.

## vanilla_bot.py

Этот файл отвечает за командную строку и RabbitMQ:

- разбирает параметры запуска;
- подключается к RabbitMQ;
- создаёт четыре временные очереди для статусов, сделок, тиков и стаканов;
- ждёт нужные рыночные данные для `slippage` или `best_quote`;
- отправляет одну лимитную заявку;
- ждёт `orderId` и финальный статус `Filled` или `Cancelled`;
- собирает сделки по `orderId`;
- печатает итоговую строку.

Очереди имеют уникальные имена с `owner`, ограничение `x-max-length = 1000` и политику `drop-head`. В конце работы consumers отменяются, очереди удаляются, соединение закрывается.

```text
CLI -> trading_core -> RabbitMQ -> статусы и сделки -> итоговая строка
```
